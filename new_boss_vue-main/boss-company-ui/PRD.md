## 一、产品概述

### 1.1 产品背景

随着互联网招聘市场的快速发展，企业 HR 面临着以下痛点:

1. **效率低下**: 传统招聘流程繁琐，简历筛选耗时耗力
2. **渠道分散**: 多个招聘平台切换，管理成本高
3. **数据缺失**: 缺乏系统化的数据分析，难以优化招聘策略
4. **沟通不便**: 与候选人沟通方式单一，响应速度慢

为解决这些问题，特开发此企业端招聘管理系统，提供一站式招聘解决方案。

### 1.2 产品愿景

打造国内领先的企业招聘管理平台，让招聘更高效、更智能、更人性化。

### 1.3 产品目标

#### 业务目标
- 提升 HR 招聘效率 50% 以上
- 提高人岗匹配准确率至 85%+
- 缩短招聘周期 30%
- 提升候选人满意度至 90%+

#### 用户目标
- 快速发布和管理职位
- 精准筛选合适简历
- 便捷与候选人沟通
- 实时掌握招聘数据

### 1.4 目标用户

#### 主要用户群体
| 用户类型 | 典型场景 | 核心需求 |
|----------|----------|----------|
| HR 经理 | 管理整个招聘流程，分配任务 | 全局把控、数据分析、团队协作 |
| 招聘专员 | 日常筛选简历、沟通候选人 | 高效筛选、便捷沟通、流程指引 |
| 业务面试官 | 参与面试评估 | 查看简历、面试安排、反馈提交 |

#### 用户规模预估
- 初期：1000+ 企业用户
- 中期：10000+ 企业用户
- 长期：100000+ 企业用户

### 2.2 核心功能详细说明

#### F001 - 用户登录

**功能描述**: 
企业用户通过账号密码验证身份，进入系统。

**优先级**: P0 - 必须实现

**用户故事**:
> 作为 HR 用户，我希望通过账号密码快速登录系统，以便开始我的招聘工作。

**功能详述**:

##### 输入要求
| 字段 | 类型 | 必填 | 验证规则 | 说明 |
|------|------|------|----------|------|
| 账号 | String | 是 | 5-50 字符 | 手机号或邮箱 |
| 密码 | String | 是 | ≥6 字符 | 加密显示 |
| 记住账号 | Boolean | 否 | - | 默认不勾选 |

##### 处理逻辑
```
1. 用户输入账号密码
2. 前端验证格式
3. 点击登录按钮
4. 调用后端 /api/login 接口
5. 后端验证身份
6. 返回 token 和用户信息
7. 前端存储到 localStorage
8. 跳转到 /company/dashboard
```

##### 输出结果
**成功场景**:
- 提示"登录成功"
- 跳转首页
- 显示欢迎语

**失败场景**:
- 账号不存在："账号或密码错误"
- 密码错误："账号或密码错误"
- 网络错误："网络错误，请稍后重试"
- 账号被封禁："账号已被封禁，请联系客服"

##### 验收标准
- [ ] 账号为空时，提示"请输入账号"
- [ ] 密码少于 6 位，提示"密码长度不能少于 6 位"
- [ ] 登录成功后，localStorage 中存在 companyToken
- [ ] 刷新页面，保持登录状态
- [ ] 未登录访问受保护路由，重定向到登录页
- [ ] 登录成功率 ≥ 99%

##### 异常处理
| 异常类型 | 处理方式 |
|----------|----------|
| 网络超时 | 提示"请求超时，请检查网络连接" |
| 服务器错误 | 提示"服务器错误，请稍后重试" |
| Token 过期 | 自动跳转登录页，提示"登录已过期" |

**功能描述**:

**优先级**: P0 - 必须实现

│  管理收到的所有简历  [批量下载][人才库]│

│  [筛选区]                           │
│  关键字 | 期望职位 | 学历 | 经验 | 状态│
├─────────────────────────────────────┤
│  [简历列表]                         │
│  ┌───┬───────┬───────┬────┬───────┐│
│  │ID │候选人 │期望职位│状态│匹配度 ││
│  ├───┼───────┼───────┼────┼───────┤│

##### 核心功能点

**1. 智能匹配度算法**
```
匹配度 = f(职位要求，简历信息)

权重分配:
- 技能匹配度：40%
- 工作经验：25%
- 学历要求：15%
- 行业背景：10%
- 其他因素：10%

输出：0-100 的百分制分数
```

**2. 匹配度可视化**
| 分数段 | 颜色 | 进度条 | 说明 |
|--------|------|--------|------|
| 90-100 | 🟢 绿色 | ████████░░ | 高度匹配 |
| 80-89  | 🟠 橙色 | ██████░░░░ | 较为匹配 |
| <80    | 🔴 红色 | ████░░░░░░ | 匹配度低 |

**3. 多维度筛选**
- 关键字：姓名、期望职位、当前公司
- 期望职位：下拉选择（前端/产品/Java 等）
- 学历：大专/本科/硕士/博士
- 经验：应届生/1-3 年/3-5 年/5 年以上
- 状态：新投递/已查看/待沟通/已沟通/不合适

**4. 简历状态流转**
```
新投递 → 已查看 → 待沟通 → 已沟通 → 不合适
         ↓                        ↑
         └────────────────────────┘
```

#### F501 - 数据看板核心指标

**功能描述**: 
展示招聘核心数据指标，帮助 HR 了解招聘效果。

**优先级**: P0 - 必须实现

**用户故事**:
> 作为 HR 经理，我希望实时查看招聘数据，以便评估招聘效果和调整策略。

**功能详述**:

##### 指标定义

**1. 职位曝光量**
- 定义：所有职位被求职者查看的总次数
- 统计周期：近 7 天/近 30 天/全部
- 更新频率：实时

**2. 主动沟通数**
- 定义：HR 主动联系求职者的次数
- 统计周期：近 7 天/近 30 天/全部
- 更新频率：实时

**3. 收到简历数**
- 定义：收到的简历总数
- 统计周期：近 7 天/近 30 天/全部
- 更新频率：实时

**4. 安排面试数**
- 定义：已安排的面试场次
- 统计周期：近 7 天/近 30 天/全部
- 更新频率：实时

##### 指标计算
```
环比增长率 = (本期数 - 上期数) / 上期数 × 100%

示例:
今日沟通数：48
昨日沟通数：40
环比增长：(48-40)/40 × 100% = +20%
```

##### 展示形式
```
┌─────────────────────────────────────────┐
│  📊 职位曝光量                           │
│  ╔════════╗                             │
│  ║  📄    ║  125,678                    │
│  ╚════════╝  +12.5% ↑                   │
└─────────────────────────────────────────┘
```

##### 视觉规范
- 卡片背景：白色
- 图标背景：渐变色（紫色系）
- 数值字体：24px，加粗
- 趋势字体：12px，绿色表示增长

##### 数据刷新
- 首次加载：立即刷新
- 定时刷新：每 5 分钟
- 手动刷新：下拉刷新

##### 验收标准
- [ ] 四个指标正确显示
- [ ] 数值格式化（千分位）
- [ ] 趋势箭头方向正确
- [ ] 百分比计算准确
