## 一、产品概述

### 1.1 产品背景

随着 BOSS 直聘平台规模的快速增长，面临以下挑战:

1. **企业真实性**: 需要严格审核企业资质，防止虚假企业
2. **职位合规性**: 需要监管职位信息，打击虚假招聘
3. **用户安全**: 需要识别和处理违规行为，保护求职者权益
4. **平台治理**: 需要完善的运营工具，提高管理效率

为解决这些问题，特开发此平台运营管理端系统。

### 1.2 产品愿景

打造智能化、高效化的平台运营管理系统，确保平台健康有序发展。

### 1.3 产品目标

#### 业务目标
- 企业审核通过率 ≥ 95%
- 违规内容处理及时率 ≥ 98%
- 用户投诉响应时间 < 2 小时
- 平台风险事件下降 50%

#### 用户目标
- 快速完成企业审核
- 便捷处理违规行为
- 全面掌握平台数据
- 高效的协同工作

### 1.4 目标用户

#### 主要用户群体
| 用户类型 | 典型场景 | 核心需求 |
|----------|----------|----------|
| 运营管理员 | 审核企业认证申请 | 高效审核、准确判断 |
| 内容审核员 | 处理高风险会话 | 快速识别、标准处置 |
| 客服管理员 | 处理用户举报 | 及时响应、公正处理 |
| 系统管理员 | 配置权限和角色 | 灵活配置、安全管理 |

#### 用户规模预估
- 初期：50-100 人运营团队
- 中期：200-500 人运营团队
- 长期：1000+ 人运营团队

### 2.2 核心功能详细说明

#### F001 - 管理员登录

**功能描述**: 
平台管理员通过账号密码验证身份，进入管理系统。

**优先级**: P0 - 必须实现

**用户故事**:
> 作为平台管理员，我希望通过账号密码快速登录系统，以便开始我的运营管理工作。

**功能详述**:

##### 输入要求
| 字段 | 类型 | 必填 | 验证规则 | 说明 |
|------|------|------|----------|------|
| 账号 | String | 是 | 3-20 字符 | 管理员账号 |
| 密码 | String | 是 | ≥6 字符 | 加密显示 |

##### 处理逻辑
```
1. 用户输入账号密码
2. 前端验证格式
3. 点击登录按钮
4. 调用后端 /api/admin/login 接口
5. 后端验证身份和权限
6. 返回 token 和管理员信息
7. 前端存储到 localStorage
8. 跳转到 /admin/dashboard
```

##### 输出结果
**成功场景**:
- 提示"登录成功"
- 跳转数据概览页
- 显示管理员欢迎语

**失败场景**:
- 账号不存在："账号或密码错误"
- 密码错误："账号或密码错误"
- 无权限："该账号无管理权限"
- 网络错误："网络错误，请稍后重试"

##### 验收标准
- [ ] 账号为空时，提示"请输入账号"
- [ ] 密码少于 6 位，提示"密码长度不能少于 6 位"
- [ ] 登录成功后，localStorage 中存在 token
- [ ] 刷新页面，保持登录状态
- [ ] 未登录访问受保护路由，重定向到登录页
- [ ] 登录成功率 ≥ 99%

#### F202 - 企业认证审核

**功能描述**: 
审核企业提交的认证申请，包括资质材料审查和工商信息核验。

**优先级**: P0 - 必须实现

**用户故事**:
> 作为运营管理员，我希望高效审核企业认证申请，以便确保平台企业信息的真实性。

**功能详述**:

##### 页面布局
```
┌─────────────────────────────────────────────┐
│  企业认证审核 - 某某信息科技有限公司         │
├──────────────┬──────────────────────────────┤
│              │                              │
│  资质材料    │   认证信息                   │
│  (图片展示)  │   (表单展示)                 │
│              │                              │
│  工商核验    │   审核操作                   │
│  (结果列表)  │   (通过/拒绝)                │
│              │                              │
└──────────────┴──────────────────────────────┘
```

##### 资质材料审查
**必需材料**:
| 材料 | 格式 | 大小 | 说明 |
|------|------|------|------|
| 营业执照 | JPG/PNG | <5MB | 必须清晰完整 |
| 法人身份证正面 | JPG/PNG | <5MB | 人像面 |
| 法人身份证反面 | JPG/PNG | <5MB | 国徽面 |
| 组织机构代码证 | JPG/PNG | <5MB | 可选 |

**审查要点**:
- [ ] 图片清晰度（文字可辨认）
- [ ] 证件完整性（四角齐全）
- [ ] 信息一致性（与填写信息匹配）
- [ ] 有效期（证件未过期）

##### 工商信息核验
**自动核验项**:
| 核验项 | 数据源 | 匹配规则 |
|--------|--------|----------|
| 统一社会信用代码 | 工商数据库 | 完全匹配 |
| 企业名称 | 工商数据库 | 完全匹配 |
| 法人代表 | 工商数据库 | 姓名匹配 |
| 注册资本 | 工商数据库 | 允许误差 |
| 成立日期 | 工商数据库 | 完全匹配 |
| 登记状态 | 工商数据库 | 存续/在业为正常 |
| 经营异常 | 工商数据库 | 无记录为正常 |

**核验结果**:
```json
{
  creditCodeValid: true,      // 信用代码有效
  companyNameMatch: true,     // 名称匹配
  legalPersonMatch: true,     // 法人匹配
  registrationStatusNormal: true, // 注册状态正常
  noAbnormalOperation: false, // 无经营异常
  riskWarning: "发现 1 条历史行政处罚记录"
}
```

##### 审核决策
**通过条件**（全部满足）:
- ✅ 资质材料清晰完整
- ✅ 工商信息核验通过
- ✅ 无重大风险记录

**拒绝条件**（满足任一）:
- ❌ 资质材料模糊不清
- ❌ 资质材料造假
- ❌ 工商信息严重不符
- ❌ 企业被列入失信名单
- ❌ 存在重大违法违规记录

##### 拒绝原因选项
**快速选择**（常用原因）:
1. 营业执照模糊不清，无法辨认
2. 证件信息与填写信息不符
3. 证件已过期
4. 疑似伪造证件
5. 企业被列入经营异常名录

**手动输入**（其他原因）:
- 支持自定义输入（100-500 字）

##### 审核流程
```
1. 打开审核页面
2. 查看资质材料（支持放大）
3. 查看工商核验结果
4. 评估风险点
5. 做出审核决策
6. 如拒绝，选择/填写原因
7. 提交审核结果
8. 系统更新企业状态
9. 发送通知给企业
```

##### 数据模型
```javascript
// 审核提交数据
{
  companyId: '10002',           // 企业 ID
  auditResult: 'approve',       // approve/reject
  rejectReason: '',             // 拒绝原因（拒绝时必填）
  remark: ''                    // 备注
}
```

##### 验收标准
- [ ] 资质材料图片清晰展示
- [ ] 图片放大功能正常
- [ ] 工商核验结果准确
- [ ] 风险点明确提示
- [ ] 审核操作流程顺畅
- [ ] 拒绝原因快速选择
- [ ] 审核结果正确保存

#### F401 - 会话审核

**功能描述**: 
对高风险的在线沟通会话进行质检和处理。

**优先级**: P1 - 应该实现

**用户故事**:
> 作为风控专员，我希望快速识别和处理高风险会话，以便维护平台沟通环境的安全。

**功能详述**:

##### 风险识别来源
**1. 敏感词检测**:
- 色情低俗词汇
- 政治敏感词汇
- 诈骗引流词汇
- 歧视侮辱词汇
- 广告推销词汇

**2. 用户举报**:
- 收到用户举报
- 多次被举报

**3. 模型识别**:
- AI 风险识别模型
- 异常行为检测

**4. 异常行为**:
- 短时间内大量发送相似消息
- 频繁添加求职者好友
- 发布虚假信息

##### 风险等级定义
| 等级 | 分数 | 颜色 | 说明 | 处理方式 |
|------|------|------|------|----------|
| 高 | 80-100 | 🔴 红色 | 严重违规 | 立即处理 |
| 中 | 50-79 | 🟠 橙色 | 可疑行为 | 优先审核 |
| 低 | 0-49 | 🟡 黄色 | 轻微异常 | 抽样检查 |

##### 审核列表
**列表字段**:
| 字段 | 说明 | 宽度 |
|------|------|------|
| # | 序号 | 60px |
| 会话 ID | 会话唯一标识 | 160px+ |
| 企业 | HR 所属企业 | 160px+ |
| 求职者 | 候选人姓名 | 120px |
| 风险等级 | 高/中/低 | 120px |
| 命中时间 | 触发风控时间 | 180px |
| 操作 | 查看详情/标记 | 200px |

##### 会话详情
**展示内容**:
- 完整聊天记录
- 敏感词高亮显示
- 风险点标注
- 会话双方信息
- 举报信息（如有）

**操作项**:
- 标记无风险
- 警告对方
- 限制聊天功能
- 临时封禁
- 永久封禁
- 加入黑名单

##### 处罚措施
| 措施 | 严重程度 | 有效期 | 适用场景 |
|------|----------|--------|----------|
| 警告 | ⭐ | 永久 | 轻微违规 |
| 限制聊天 | ⭐⭐ | 7 天 | 中度违规 |
| 临时封禁 | ⭐⭐⭐ | 30 天 | 严重违规 |
| 永久封禁 | ⭐⭐⭐⭐ | 永久 | 特别严重违规 |
| 加入黑名单 | ⭐⭐⭐⭐⭐ | 永久 | 屡教不改 |

##### 审核流程
```
系统识别风险
    ↓
加入审核队列
    ↓
显示在会话审核列表
    ↓
审核员点击查看详情
    ↓
查看完整聊天记录
    ↓
判断是否违规
    ↓
【无违规】标记无风险 → 结束
    ↓
【有违规】选择处罚措施
    ↓
确认处罚
    ↓
执行处罚 → 结束
```

- [ ] 敏感词高亮显示
- [ ] 处罚措施可选
- [ ] 审核结果可追溯
- [ ] 处罚执行成功

## 六、风险与应对

### 6.1 技术风险

#### 风险 1: 工商数据接口不稳定
**概率**: 中  
**影响**: 高  
**应对措施**:
- 接入多个数据源备份
- 本地缓存核验结果
- 手动核验流程备选

#### 风险 2: 图片加载慢
**概率**: 高  
**影响**: 中  
**应对措施**:
- CDN 加速
- 图片压缩优化
- 懒加载

### 6.2 业务风险

#### 风险 1: 审核标准不统一
**概率**: 高  
**影响**: 中  
**应对措施**:
- 制定详细审核标准
- 定期培训审核人员
- 建立审核质量抽查机制

#### 风险 2: 恶意投诉
**概率**: 中  
**影响**: 低  
**应对措施**:
- 建立恶意投诉识别机制
- 完善申诉流程
- 保留证据链
